## 파일 관리 원칙

### 실제 파일 저장 위치
```
public/docs/certs/         ← 웹에서 다운로드 가능한 파일 (Supabase Storage 연동 전 임시)
```
Supabase Storage 연동 후에는 `downloads` 버킷으로 이전 예정.

### 파일명 규칙
```
[카테고리코드]_[문서유형]_[대상모델또는내용]_[발급기관약칭]_[취득연도].pdf

예시:
  CERT_KMARK_SUR-2400P_KETI_2020.pdf
  PAT_특허_제10-1028910호_원적외선난방장치.pdf
  EU_CE_SUR-1200_2023.pdf
  TEST_IP55_방수시험성적서_KETI_2021.pdf
```

### 카테고리 코드표

| 코드 | 분류 | 설명 |
|------|------|------|
| `BIZ` | 사업·공장 | 사업자등록증, 공장등록증명서 등 |
| `GOV` | 조달·정부 인증 | 조달청 우수제품, 혁신제품, 표창장 등 |
| `CERT` | 성능·품질 인증 | K마크, ISO, 전기용품인증 등 |
| `SAFE` | 안전·특수 인증 | 방폭, 방수(IP) 등 |
| `PAT` | 특허·디자인 | 국내 특허, 디자인 등록 |
| `TEST` | 시험 성적서 | 원적외선 방사율, 항균, 탈취율 등 |
| `EU` | 해외 인증 | CE, RoHS 등 유럽 인증 |
| `SPEC` | 기술 자료 | 시방서, 시공상세도 등 |

## 파일 업로드 체크리스트

파일을 올릴 때는 아래 순서로 진행:

1. **파일명 규칙 확인** — 위 파일명 칸의 이름 그대로 저장
2. **Supabase Storage 업로드** — `downloads` 버킷 > 카테고리 폴더에 업로드
   ```
   downloads/
   ├── BIZ/
   ├── GOV/
   ├── CERT/
   ├── SAFE/
   ├── PAT/
   ├── TEST/
   ├── EU/
   └── SPEC/
   ```
3. **CatalogPage 업데이트** — `src/pages/resources/CatalogPage.tsx`의 해당 항목 `ready: true`, URL 추가
4. **이 파일의 "파일 있음" 칸을 ☑ 로 변경**

## 현재 CertificationsPage 표기 vs 실제 현황

| 항목 | 현재 페이지 표기 | 실제 현황 | 수정 필요 |
|------|-----------------|-----------|-----------|
| K마크 범위 | SUR-D300P ~ SUR-3600 | SUR-D300P, 600P, 1200P, 1800P, 2400P, 3600P (6종) | ✅ 일치 |
| 방수 등급 | IP-65 | IP-55 (성적서 기준) | ⚠️ 확인 후 수정 |
| 특허 수 | 4+ | **10건 (특허 9 + 디자인 1)** | ⚠️ 수정 필요 |
| ISO 취득 기관 | 미표기 | 확인 필요 | — |
| 표창장 | 미표기 | 2건 (2016, 2019) | 추가 가능 |
