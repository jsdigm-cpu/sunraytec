# AI HANDOFF PROMPTS

이 문서는 클로드 채팅, 코덱스, 클로드 코드에 공통으로 붙여넣을 수 있는 시작 프롬프트와 작업 템플릿을 모아둔 파일입니다.

## 1. 공통 프로젝트 소개 프롬프트

```text
이 프로젝트는 썬레이텍 회사 홈페이지입니다.
기획 기준 문서는 `sunraytec_website_plan_v2.html` 입니다.
현재 상태 기준 문서는 `PROJECT_STATUS.md` 입니다.
지금 해야 할 일 기준 문서는 `NEXT_TASK.md` 입니다.
중요한 결정사항은 `DECISIONS.md` 에 기록합니다.

작업 전에는 반드시 현재 코드와 위 문서들을 먼저 읽고,
문서와 코드가 다르면 코드 기준의 현재 상태를 먼저 짧게 요약해주세요.

미구현 기능을 이미 구현된 것처럼 가정하지 말고,
실제 구현 가능 여부는 반드시 현재 코드 기준으로 판단해주세요.
한 번에 한 작업만 다뤄주세요.
```

## 2. 클로드 채팅용 시작 프롬프트

```text
너는 이 프로젝트에서 기획 및 작업 요청문 작성 담당이야.
코드 구현 자체를 단정하지 말고, 현재 문서와 코드 상태를 존중하는 방향으로 작업 요청문을 정리해줘.

역할:
- 화면 구조, 문구, IA, 기능 우선순위 정리
- 구현 AI에게 전달할 작업 요청문 초안 작성
- 작업을 큰 덩어리로 말하지 말고 작은 작업 단위로 분해

주의:
- 구현됐는지 확실하지 않은 기능은 단정하지 말 것
- 작업 요청문은 1개 작업 단위로 쓸 것
- 문서에 남기기 쉬운 표현으로 정리할 것
```

## 3. 코덱스용 시작 프롬프트

```text
너는 이 프로젝트에서 구현, 코드 검토, 구조 정리 담당이야.
먼저 `PROJECT_STATUS.md`, `NEXT_TASK.md`, `DECISIONS.md`, 관련 코드 파일을 읽고
현재 상태와 이번 작업 범위를 짧게 요약한 다음 작업해줘.

작업 규칙:
- 문서와 코드가 다르면 코드 기준으로 차이를 먼저 보고
- 한 번에 한 작업만 수행
- 작업 후에는 변경 내용, 테스트 여부, 남은 리스크를 짧게 정리
- 가능하면 다음 추천 작업 1~2개도 함께 제안
```

## 4. 클로드 코드용 시작 프롬프트

```text
이 프로젝트는 썬레이텍 회사 홈페이지입니다.
작업 전 `PROJECT_STATUS.md`, `NEXT_TASK.md`, `DECISIONS.md`, 관련 코드 파일을 먼저 읽고,
현재 상태와 이번 작업 범위를 짧게 요약해주세요.

규칙:
- 실제 코드 기준으로 판단할 것
- 미구현 기능을 이미 있는 것처럼 전제하지 말 것
- 한 번에 한 작업만 수정할 것
- 작업 후 변경 파일, 테스트 여부, 남은 이슈를 짧게 정리할 것
```

## 5. 코드 수정 요청 템플릿

```text
다음 작업 1개만 진행해주세요.

[작업명]

목표:
[무엇을 바꾸고 싶은지]

완료조건:
- [조건 1]
- [조건 2]
- [조건 3]

관련 문서:
- PROJECT_STATUS.md
- NEXT_TASK.md
- DECISIONS.md

관련 코드:
- [파일 경로 1]
- [파일 경로 2]

작업 전에는 현재 상태를 짧게 요약하고,
작업 후에는 변경 내용 / 테스트 여부 / 남은 리스크를 짧게 알려주세요.
```

## 6. 코드 리뷰 요청 템플릿

```text
현재 코드 변경분을 리뷰해주세요.

리뷰 기준:
- 버그 가능성
- 라우트/링크 불일치
- 상태 관리 연결 문제
- 데이터 구조 중복
- 지금 단계에서 과한 구현 여부

중요:
- 요약보다 finding을 먼저 써주세요.
- 심각도 높은 것부터 정리해주세요.
- 코드와 문서가 어긋나는 부분이 있으면 함께 알려주세요.
```

## 7. 현재 상태 점검 요청 템플릿

```text
현재 프로젝트 상태를 점검해주세요.

반드시 아래 순서로 봐주세요:
1. PROJECT_STATUS.md
2. NEXT_TASK.md
3. DECISIONS.md
4. 실제 코드

알고 싶은 것:
- 지금 실제로 구현된 범위
- 문서와 코드가 다른 부분
- 다음 작업으로 바로 들어가기 좋은 1~3개 항목
- 지금 배포하면 위험한 부분
```

## 8. 사용자 운영 순서 템플릿

### 작업 시작 전
1. `NEXT_TASK.md`에서 지금 할 작업 1개를 고른다.
2. 필요하면 클로드 채팅으로 작업 요청문을 다듬는다.
3. 공통 프로젝트 소개 프롬프트 + 작업 요청 템플릿을 구현 AI에 전달한다.

### 작업 완료 후
1. 구현 AI의 결과 요약을 읽는다.
2. 완료된 내용은 `PROJECT_STATUS.md`에 반영한다.
3. 새로 결정된 기준이 있으면 `DECISIONS.md`에 추가한다.
4. 다음 작업만 `NEXT_TASK.md`에 남긴다.
5. 작업 도중 세션이 끊길 가능성이 있으면 `SESSION_HANDOFF.md`를 갱신한다.

## 9. 초보자용 아주 짧은 사용 버전

### 클로드 채팅에 넣을 말
```text
PROJECT_STATUS.md와 NEXT_TASK.md를 기준으로,
지금 해야 할 작업 1개를 구현 AI에게 전달할 요청문으로 짧게 정리해줘.
```

### 코덱스 또는 클로드 코드에 넣을 말
```text
PROJECT_STATUS.md, NEXT_TASK.md, DECISIONS.md와 관련 코드 파일을 먼저 읽고,
현재 상태를 짧게 요약한 뒤 NEXT_TASK.md의 현재 최우선 작업 1개만 진행해주세요.
```

## 10. 세션 중단 대비 인계 프롬프트

### 작업 중간 저장 요청
```text
세션이 끊겨도 이어갈 수 있게 현재 작업 상태를 `SESSION_HANDOFF.md`에 반영해주세요.

꼭 포함할 내용:
- 방금 끝낸 것
- 아직 진행 중인 것
- 마지막으로 확인한 파일
- 다음 AI가 바로 해야 할 1개 작업
- 남은 리스크 또는 막힌 점
```

### 다음 AI에게 이어달라고 할 때
```text
`PROJECT_STATUS.md`, `NEXT_TASK.md`, `DECISIONS.md`, `SESSION_HANDOFF.md`를 먼저 읽고,
직전 세션이 어디까지 했는지 5줄 이내로 요약한 뒤 다음 작업 1개만 이어서 진행해주세요.
```
