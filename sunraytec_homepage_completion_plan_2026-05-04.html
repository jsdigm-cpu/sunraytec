Sunraytec Completion Plan

썬레이텍 홈페이지 완성 실행 플랜

이 문서는 리뷰 보고서를 바탕으로 현재 홈페이지를 공개·영업·조달 상담에 활용 가능한 수준까지 끌어올리기 위한 실행 계획입니다. 다른 AI가 이어서 작업하더라도 최우선순위는 이 문서를 기준으로 잡습니다.

작성일: 2026-05-04 개발 지침: HOMEPAGE_COMPLETION_GUIDE.md 검증 기준: lint + build + 모바일 QA 최우선: 모바일·조달문구·SEO·lint

1. 완성 전략 요약

지금 필요한 것은 새 기능을 무작정 늘리는 것이 아니라, 신뢰를 해치는 문제를 먼저 제거하고 본문을 영업형 구조로 다시 정리하는 것입니다.

1순위

신뢰 리스크 제거

모바일 잘림, 조달 단정 문구, SEO 도메인 불일치, lint 실패부터 해결합니다.

2순위

홈 본문 재구성

카드 반복을 줄이고 구매자별 진입, 현장 문제, 검증 수치, 제품 선택 흐름을 만듭니다.

3순위

핵심 페이지 보강

제품, 기술, 시공사례, 문의, 자료실을 실제 영업 흐름에 맞게 다듬습니다.

4순위

운영 백엔드 고도화

문의 알림, 자료 권한, 다운로드 로그, 관리자 이력을 추가합니다.

2. 최우선 작업 목록

다른 AI가 투입되면 이 표의 순서대로 처리해야 합니다.

우선순위 작업 대상 완료 기준
P0 모바일 가로 잘림 수정 홈, 우수제품, 견적문의, 복사난방 원리, SubHero 공통 390px/430px/768px/1440px에서 가로 스크롤과 텍스트 잘림 없음
P0 조달/계약 문구 정확성 정리 홈 제품 라인업, 우수제품, MAS, 제품 상세 출시 시점 계약 완료·종합쇼핑몰 등재 기준으로 작성하고, 공개 직전 실제 상태 최종 검수
P1 SEO 대표 도메인 정리 PageSEO, JsonLd, sitemap, robots, index.html canonical/OG/sitemap 도메인을 현재 운영 도메인 www.sunraytec.co.kr 기준으로 정리
P1 TypeScript lint 실패 해결 sharp 의존성, 이미지 스크립트, tsconfig 필요 여부 npm run lint, npm run build 모두 통과
P2 제품 식별번호/계약상태 검수 products.ts, 제품 카드, 제품 상세, sitemap 제품별 식별번호, MAS/우수제품 상태, 추가구성품 향후 등록 과제가 표준화

3. 단계별 실행 로드맵

작업은 아래 단계로 나누어 진행합니다. 앞 단계가 끝나기 전에는 뒤 단계의 디자인 욕심을 크게 내지 않는 편이 좋습니다.

Phase 0

작업 전 안전 확인

현재 변경사항과 빌드 상태를 먼저 확인합니다.

  • git status --short --branch
  • npm run lint
  • npm run build
  • 기존 변경사항 되돌리지 않기
Phase 1

신뢰 리스크 제거

모바일, 조달 표현, SEO, lint 문제를 먼저 해결합니다.

  • SubHero/HeroSection 모바일 텍스트 잘림 수정
  • 복사난방 원리와 솔루션 다이어그램 모바일 1열 대응
  • 우수제품은 출시 시점 계약 완료·종합쇼핑몰 등재 기준으로 콘텐츠 작성
  • 공개 직전 계약/등재/식별번호 최종 검수 항목 유지
  • 대표 도메인 www.sunraytec.co.kr 기준으로 canonical/sitemap 일괄 정리
Phase 2

홈 본문 구조 재설계

카드 반복형 홈페이지에서 구매자별 설득형 홈페이지로 전환합니다.

  • 공공기관/산업현장/설계자/파트너 빠른 진입
  • 현장 문제 Before/After
  • 시험기관·성적서 번호 포함 검증 수치 바
  • 제품 선택 매트릭스와 조달 구매 흐름
Phase 3

제품·기술·사례 페이지 완성

방문자가 검토자료와 문의로 자연스럽게 이동하도록 핵심 페이지를 보강합니다.

  • 제품 상세에 카탈로그/시방서/인증서 연결
  • 기술 주장마다 근거 자료 링크 추가
  • 시공사례 템플릿 통일 및 실제 사진 확대
  • 문의 폼은 박이사님 담당, master@sunraytec.net 수신 기준으로 정리
  • 자료실 모바일/권한 구조는 자료 정리 이후 후순위로 진행
Phase 4

백엔드 운영 고도화

홈페이지를 운영 가능한 영업 도구로 만듭니다.

  • 문의 접수 알림과 상태 이력
  • 자료 정리 후 다운로드 권한과 로그
  • 관리자 수정/삭제 audit log
  • 전화, 견적, 나라장터, 다운로드 이벤트 분석
Phase 5

출시 전 QA

실제 고객이 보는 화면과 검색엔진 기준을 마지막으로 확인합니다.

  • 주요 페이지 데스크톱/태블릿/모바일 캡처
  • 제품 이미지, CTA, 문의 폼, 다운로드 흐름 확인
  • Google Search Console, Naver Search Advisor 등록 준비
  • Vercel Preview/Production 확인

4. 홈 본문 권장 구성

박이사님이 느끼신 딱딱함을 줄이기 위한 핵심은 섹션 모양을 바꾸는 것보다 방문자의 의사결정 흐름을 바꾸는 것입니다.

01

구매자별 빠른 진입

공공기관, 산업현장, 설계/감리, 파트너가 자기 상황에 맞는 페이지로 바로 들어가게 합니다.

02

현장 문제 Before/After

고천장 열 손실, 분진 비산, 결로, 소음 문제를 실제 사진과 다이어그램으로 설명합니다.

03

공식 검증 수치

KTR, KGS, K마크, ISO, 조달청 지정 등 수치를 출처와 함께 보여줍니다.

04

제품 선택 매트릭스

면적, 천장고, 설치방식, 계약구분으로 적합 모델을 찾게 합니다.

05

대표 시공사례

현장 사진, 설치 모델, 설치 전 문제, 결과 수치를 한 카드에서 보여줍니다.

06

조달 구매 안내

MAS 9개 식별번호와 우수제품 계약 완료 기준 구매 흐름을 안내하되, 공개 직전 등재 상태를 재검수합니다.

5. 페이지별 완성 목표

각 페이지가 맡아야 할 역할입니다.

페이지 목표 핵심 보완
홈 첫 방문자를 구매자 유형별로 안내 카드 반복 축소, 현장 사진, 검증 수치, 제품 선택 흐름 추가
제품안내 제품별 계약상태와 적용처를 명확히 비교 식별번호 검수, 계약상태 배지, 상세자료 링크
기술·솔루션 광고가 아닌 검토자료처럼 신뢰 확보 시험성적서 번호, 인증서 링크, 적용 모델 연결
시공사례 실제 납품 경험과 효과 증명 실제 사진, 설치 모델, 설치 대수, 결과 수치
문의 견적/자료 요청을 쉽게 접수 모바일 폼, 개인정보 안내, 접수 알림
자료실 공공기관/설계자/파트너용 자료 제공 공개/승인 자료 구분, 다운로드 로그, 권한 관리
