1. 총평

현재 사이트는 “홈페이지 초안” 수준은 넘었고, 제품·조달·기술·시공사례·관리자 기능까지 담은 사업형 웹사이트의 골격은 갖췄습니다.

잘된 점

조달·기술 중심 포지셔닝이 명확함

일반 난방 제품 쇼핑몰이 아니라 공공조달, 산업현장, 특수환경을 겨냥한 B2B/B2G 사이트라는 방향은 좋습니다.

아쉬운 점

본문 섹션이 “카드 묶음”으로 반복됨

히어로는 힘이 있는데, 본문은 카드 3~4개와 설명문 위주가 반복되어 제품의 실제감과 현장감이 약합니다.

위험한 점

모바일 잘림과 조달 문구 정확성

모바일에서 긴 제목과 태그가 오른쪽으로 잘리고, 우수제품 제3자단가·수의계약 문구는 현재 실제 계약 상태와 반드시 맞춰야 합니다.

가장 먼저 고칠 것은 디자인의 미세한 취향이 아니라 모바일 가로 잘림, 조달 관련 표현, 도메인/SEO 불일치, TypeScript lint 실패입니다. 이 네 가지는 방문자의 신뢰와 운영 안정성에 직접 영향을 줍니다.

2. 우선순위 이슈

바로 작업 순서로 바꿔도 되는 핵심 이슈입니다.

우선순위 이슈 근거 개선 방향

3. UI 디자인 리뷰

박이사님 말씀처럼 상단 히어로는 꽤 정리됐습니다. 문제는 본문에서 같은 문법이 반복된다는 점입니다.

강점

히어로와 대메뉴의 첫인상은 힘이 있음

실제 설치 사진을 배경으로 쓰고, 조달·인증 배지를 전면에 노출한 방향은 썬레이텍의 업종 특성과 잘 맞습니다. 상단 메뉴도 제품, 기술, 시공사례, 자료실, 고객센터로 구매 여정을 어느 정도 반영합니다.

약점

본문이 “텍스트 카드 + 아이콘”으로 평평해짐

홈의 핵심 지표, 4대 검증 포인트, 업종별 솔루션, 제품 라인업, 인증 섹션이 모두 유사한 카드 배열입니다. 읽는 사람은 정보가 많다고 느끼지만, 기억에 남는 장면은 적습니다.

디자인 개선 제안

섹션 다양화

카드 그리드 비중 줄이기

본문의 40~50%는 카드가 아닌 비교 다이어그램, 현장 사진, 인증서 썸네일, 구매 흐름, 도면/설치 단면 같은 형태로 바꾸는 것이 좋습니다.

현장감

실제 제품과 설치 장면을 더 많이

복사난방 패널은 “보이는 제품”입니다. 제품 사진, 천장 설치 전후, 열이 도달하는 위치, 제어반 화면 등을 본문 중간중간 배치해야 신뢰가 생깁니다.

B2B 톤

이모지 의존 줄이기

현재는 🏛️, 🏭, 🚀 등 이모지가 많습니다. 친근함은 있지만 공공기관·설계사무소 담당자에게는 다소 가벼워 보일 수 있어 lucide 아이콘 또는 실제 문서 아이콘으로 정리하는 편이 좋습니다.

4. 콘텐츠 리뷰

콘텐츠는 “정보량”보다 “구매자별 설득 경로”가 중요합니다. 지금은 좋은 수치가 흩어져 있고, 누구에게 어떤 순서로 말하는지가 약합니다.

공공기관 담당자

조달 가능 여부가 최우선

우수제품 지정, MAS 등록, 나라장터 식별번호, 계약 방식, 제출서류 다운로드가 한 화면에서 확인되어야 합니다.

시설·공무팀

운영비와 고장 부담이 최우선

기존 온풍기 대비 전기요금, 유지관리, 필터, 소음, 작업자 체감 개선을 숫자와 사례로 보여줘야 합니다.

설계·감리 담당자

도면·시방·인증 범위가 최우선

제품별 CAD, 시방서, 시험성적서, 설치 조건, 전기 인입 조건이 빠르게 연결되어야 합니다.

현재 콘텐츠는 “우리 제품은 좋습니다”라는 설명은 많은데, “당신 상황에서는 왜 이 제품이어야 하는가”를 끝까지 끌고 가는 구조가 부족합니다. 특히 공공·교육, 산업·물류, 국방·특수, IoT 제어는 서로 다른 구매 논리를 가져야 합니다.

5. 페이지별 콘텐츠 개선안

현재 코드와 화면 흐름을 기준으로 페이지별 보완 방향을 정리했습니다.

페이지 현재 평가 개선 포인트
홈 히어로는 좋지만 이후가 수치 카드, 검증 카드, 솔루션 카드, 제품 카드로 반복됩니다. “구매자 선택 경로”를 추가하세요. 공공기관 / 산업현장 / 설계사무소 3개 진입 카드, 실제 현장 사진 기반 Before/After, 검증자료 링크형 수치 카드가 필요합니다.
우수제품·MAS 제품 카드와 조달 절차는 있으나 계약 상태와 실제 등록 상태의 정확성 검증이 필요합니다. 계약 완료 전후 문구를 분리하고, 제품별 식별번호·계약구분·등록상태·나라장터 검색어를 표준화하세요.
제품 선택 가이드 ProductGuidePage.tsx에 page별 데이터가 있지만 히어로는 모두 “제품 선택 가이드”로 동일합니다. 방폭·특수, 개인용·욕실형, 스마트 제어, 제품 비교 각각의 title/desc를 실제 페이지 상단에 반영해야 합니다.
기술·솔루션 수치와 설명은 강하지만 텍스트 밀도가 높고 출처 파일 연결이 약합니다. 각 주장 옆에 “시험성적서 보기”, “인증서 보기”, “적용 가능 제품 보기” 버튼을 붙이세요. 수치가 많을수록 출처가 바로 보여야 신뢰가 올라갑니다.
시공사례 수치형 대표 사례는 좋지만 일부는 정적 텍스트·아이콘 placeholder 느낌이 납니다. 실제 사진, 설치 전 조건, 설치 모델, 설치 대수, 결과 수치, 담당자 코멘트 형식으로 통일하세요.
견적문의 문의 유형 선택과 폼은 잘 되어 있으나 모바일 레이아웃과 개인정보 안내가 부족합니다. 모바일에서 1열 폼으로 재배치하고, 개인정보 수집 항목·보관 기간·문의 처리 흐름을 체크박스 주변에 간단히 노출하세요.

6. 홈 본문 재구성 제안

홈페이지가 덜 딱딱해지려면 섹션의 “모양”과 “역할”을 동시에 바꿔야 합니다.

1. 구매자별 진입

공공기관, 산업현장, 설계사무소/감리, 대리점 파트너로 나눠 첫 클릭 경로를 제공합니다.

2. 현장 문제 시각화

“고천장 열 손실”, “분진 비산”, “결로”, “소음”을 사진·다이어그램으로 보여줍니다.

3. 공신력 수치

KTR, KGS, K마크, ISO, 조달청 지정 등 수치를 출처 링크와 함께 카드가 아닌 증빙 바 형태로 배치합니다.

4. 제품 선택 흐름

면적·천장고·설치방식·계약구분으로 제품을 고르는 “의사결정 표”를 만듭니다.

추천 홈 흐름: 히어로 → 구매자별 빠른 진입 → 현장 문제 Before/After → 공식 검증 수치 → 공간별 솔루션 → 제품 선택 매트릭스 → 대표 시공사례 → 조달 구매 안내 → 계산기/견적 CTA.

7. 백엔드·운영 구조 리뷰

Supabase 기반의 관리자/CMS 구조는 이미 들어와 있습니다. 다만 운영 사이트로 쓰려면 “문의 처리”와 “자료 다운로드 권한”이 더 단단해야 합니다.

현재 강점

CMS와 관리자 기능의 기본 골격 보유

제품, 히어로 콘텐츠, 시공사례, 자료실, 회원, 공지, FAQ, 견적문의 관리 컴포넌트가 존재합니다. Supabase RLS와 storage bucket 정책도 마련되어 있어 초안으로는 좋은 출발입니다.

현재 리스크

운영 흐름과 보안 경계가 아직 느슨함

프론트에서 Supabase를 직접 호출하고, 문의 폼에는 스팸 방지·서버 검증·알림 자동화가 보이지 않습니다. localStorage fallback은 운영 중 DB 저장 실패를 눈치채기 어렵게 만들 수 있습니다.

백엔드 개선 우선순위

영역 현재 상태 권장 개선
문의 관리 inquiries 테이블 insert 중심 문의 접수 이메일/문자 알림, 상태 변경 이력, 담당자 배정, 스팸 방지, 서버 측 validation 추가
자료실 자료 문서 관리 컴포넌트와 storage bucket 구조 있음 공개자료/파트너자료/공공기관자료를 권한별로 분리하고 다운로드 로그를 남김
콘텐츠 관리 히어로 중심 CMS 저장, 일부는 코드 상수 핵심 페이지 문구를 DB/CMS화하되, 제품/조달 데이터는 검수 승인 절차를 둠
권한 관리 admin/partner role, pending/approved 상태 구조 있음 관리자 작업 audit log, 회원 승인 이력, 자료 다운로드 권한 만료일 추가
분석 SEO는 있으나 전환 이벤트 체계는 약함 견적 클릭, 전화 클릭, 나라장터 클릭, 자료 다운로드, 계산기 사용 이벤트를 저장

8. SEO·접근성·성능 리뷰

SEO 기본 작업은 꽤 잘 되어 있습니다. 다만 대표 도메인과 접근성, 모바일 사용성은 반드시 보완해야 합니다.

SEO 강점

PageSEO·JSON-LD·sitemap 구조 있음

페이지별 title/description, Product JSON-LD, Organization JSON-LD, sitemap, robots가 들어간 점은 좋습니다.

SEO 리스크

도메인과 OG 이미지 정리 필요

canonical이 .com으로 고정되어 있고, 카카오톡·SNS 공유용 1200×630 대표 OG 이미지가 별도로 보이지 않습니다.

성능 리스크

초기 번들에 Supabase와 Motion 부담

빌드 결과 vendor-supabase, vendor-motion, index chunk가 큽니다. 고객용 첫 화면에서는 Supabase 의존을 더 늦게 불러오는 구조를 검토할 만합니다.

공공기관·교육시설 고객을 고려하면 KWCAG 2.2 기준의 대체텍스트, 명도 대비, 키보드 이동, 초점 표시, 자동 움직임 제어도 체크해야 합니다. 현재 일부 focus style은 좋지만 모바일 잘림과 움직임 제어는 더 확인이 필요합니다.

9. 조달·인증 문구 검수 제안

썬레이텍 사이트의 신뢰는 조달 문구의 정확성에서 크게 갈립니다.

현재 표현 유형 위험 권장 표현
“조달청 3자단가”, “수의계약으로 입찰 없이 간편 구매” 계약 완료 전이면 과장 또는 사실 불일치로 보일 수 있음 “우수제품 지정 완료, 제3자단가계약 완료 후 나라장터 등록 예정” 또는 완료 후 “나라장터 종합쇼핑몰 등록 제품”

조달청 공식 안내에 따르면 우수제품은 지정 후 제3자단가계약 또는 총액계약 등 구매 절차로 이어질 수 있습니다. 따라서 “지정”과 “계약/등록 완료”는 홈페이지 문구에서 반드시 분리해야 합니다.
