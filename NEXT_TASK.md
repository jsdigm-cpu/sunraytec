# NEXT TASK

마지막 업데이트: 2026-04-21

## 사용 규칙
- 이 파일에는 항상 "지금 바로 할 일"만 1~3개 유지합니다.
- 끝난 작업은 이 파일에서 지우고 `PROJECT_STATUS.md`에 반영합니다.
- 방향이 바뀌는 작업은 시작 전에 `DECISIONS.md`를 먼저 업데이트합니다.

## 현재 최우선 작업 1
### 작업명
제품 상세 전용 페이지를 만들지, 공통 제품 페이지 유지할지 결정

### 목표
현재 우수제품 카드가 공통 제품 페이지로 연결되는 상태이므로, 이후 상세 전개 방식을 정해 링크 구조를 안정화합니다.

### 완료조건
- 제품 카드 클릭 시 목표 경로 정책이 정해진다.
- 상세 페이지가 필요하면 후속 작업 항목으로 분리된다.
- 결정 내용이 `DECISIONS.md` 또는 `NEXT_TASK.md`에 반영된다.

### 구현 전 확인사항
- 현재 실제 살아있는 제품 관련 라우트는 `/products`, `/products/excellence`다.
- 카드에서 없는 상세 경로로 보내지 않도록 이미 안전 처리돼 있다.

## 현재 최우선 작업 2
### 작업명
Admin CMS 반영 범위를 Hero 외 섹션으로 확장할지 정리

### 목표
현재 Hero까지 연결된 CMS 흐름을 어디까지 확장할지 정해서, 이후 데이터 구조와 편집 UI가 흔들리지 않게 합니다.

### 완료조건
- 다음 연결 후보 섹션이 정해진다.
- 필요한 타입/데이터 구조 확장 범위가 정리된다.
- 결정 내용이 `DECISIONS.md` 또는 후속 작업 문서에 반영된다.

### 구현 전 확인사항
- 현재 `ContentEditor`는 Hero만 수정 가능하다.
- 인증, 사례, CTA 등 다른 섹션은 아직 하드코딩 중심이다.

## 현재 최우선 작업 3
### 작업명
번들 크기 경고를 다음 단계에서 정리할지 검토

### 목표
프로덕션 빌드 경고가 기능상 문제는 아니지만, 이후 페이지가 늘어날 때 성능 리스크가 커질 수 있으므로 대응 필요성을 판단합니다.

### 완료조건
- 코드 분할이 지금 필요한지 판단된다.
- 필요하면 후속 작업으로 라우트 단위 분할 또는 lazy loading이 등록된다.
- 지금 당장 미뤄도 되는지 여부가 기록된다.

### 구현 전 확인사항
- 현재 빌드는 성공하지만 500kB 경고가 출력된다.
- 서브 페이지가 계속 늘어날 예정이라 번들 크기는 다시 커질 가능성이 있다.
